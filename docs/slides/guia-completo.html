O que é o Playwright?

O Playwright é um framework de automação da Microsoft para testes end-to-end e API — com suporte nativo a Chromium, Firefox e WebKit.

  • Escrito em TypeScript / JavaScript
  • API moderna com auto-wait — menos flakiness
  • Fixtures page, request, context por teste
  • UI Mode, trace viewer, screenshots e relatório HTML
  • Paralelismo nativo e isolamento entre workers
Diferente do Cypress, o Playwright usa WebDriver BiDi / protocolos nativos do browser e roda fora do contexto da aplicação — suportando múltiplos browsers e testes API sem UI no mesmo runner.

Por que usar Playwright?

Velocidade & paralelismo

Workers paralelos e browser contexts isolados por teste.

Locators resilientes

getByTestId, role, text — recomendados pela doc oficial.

Debug rico

UI Mode, headed, Inspector e trace on retry.

E2E + API unificados

page para UI e request para contratos REST.

test('GET /resources — Authorization header omitted', async ({ playwright }, testInfo) => {
  await test.step('New context without Authorization header', async () => {
    const ctx = await playwright.request.newContext({ baseURL })
    try {

      await attachHttpExchangeReport(testInfo, {
        label: 'resources-no-auth',
        method: 'GET',
        url: '/api/resources',

export function runPatchTests(context, { payloads, tag }) {
  for (const [label, payload] of Object.entries(payloads)) {
    test.describe(label, () => {
      test(`204 — PATCH accepted, GET profile matches`, async ({ request, authToken }) => {
        const patchRes = await request.patch(PATCH_PATH, { headers, data: payload })
        expect(patchRes.status()).toBe(204)
        const profile = await pollGetUntilMatch(request, GET_PATH, headers, payload)
        expect(profile).toMatchObject(normalizePayload(payload))
      })
    })
  }
}

PATCH → poll GET — eventual consistency

async function pollGetUntilMatch(request, path, headers, expected, deadlineMs = 10_000) {
  const start = Date.now()
  while (Date.now() - start < deadlineMs) {
    const res = await request.get(path, { headers })
    if (res.status() === 200) {
      const body = normalizeProfile(await res.json())
      if (matchesExpected(body, expected)) return body
    }
    await new Promise((r) => setTimeout(r, 500))
  }
  throw new Error('GET did not match PATCH payload within deadline')
}

export class LoginPage {
  constructor(private readonly page: Page) {}

  emailInput() { return this.page.getByTestId('login-email') }
  submitBtn()  { return this.page.getByTestId('login-submit') }

  async visit() {
    await this.page.goto('/web/login.html')
    return this
  }

  async loginWith(email: string, password: string) {

Acessibilidade — axe-core

Suite a11y — scans com @axe-core/playwright em páginas-chave. Doc: a11y.md

import AxeBuilder from '@axe-core/playwright'

async function assertNoCriticalViolations(page, scope?: string) {
  let builder = new AxeBuilder({ page }).disableRules(['color-contrast'])
  if (scope) builder = builder.include(scope)
  const results = await builder.analyze()
  const critical = results.violations.filter((v) => v.impact === 'critical')
  expect(critical).toEqual([])
}

// Data-driven: Dashboard, Login, Wizard, Components, Settings, UI States

    await loginViaApi(page, request)
    await page.goto(path)
    await assertNoCriticalViolations(page, scope)
  })
}

Boas práticas

  • Page Objects em pages/ — locators, actions e assertions isolados
  • getByTestId + data-testid — mesma convenção do testflow-cypress
  • loginViaApi no beforeEach — specs de feature sem repetir login UI
  • beforeAll + resposta compartilhada — contratos API sem N requests idênticos
  • validateSchema — tipos da resposta REST de forma declarativa
  • page.route + waitForRequest — spy payload e mock de erro
  • Fixtures JSON — dados determinísticos (credentials, team-member)
  • Um cenário por test() — falhas fáceis de diagnosticar no report
  • projects no config — 18 suites; CI roda tudo com npm test
  • test.step + attachHttpExchangeReport — debug de API no HTML report
  • EXPECT constants — status codes nomeados, não magic numbers
  • Tags @smoke / @regression — filtrar localmente com test:grep:*
  • globalSetup + cache de token — setup pesado uma vez por run
  • visitAuthenticated — navega autenticado para qualquer rota
  • AxeBuilder + @a11y — scans de acessibilidade sem violações críticas
  • toHaveScreenshot + @visual — baselines com maxDiffPixelRatio: 0.05
  • ShellPage.navItems() — navegação data-driven no app shell
  • Factories em support/factories/ — payloads wizard e PATCH reutilizáveis
  • readFixture + expectSameMembers — contratos golden sem JSON Schema
  • JsonPatchBuilder + runPatchTests() — PATCH data-driven enterprise
  • Leia o walkthrough bloco a bloco em docs/pt/ ou docs/en/
  • npm run test:ui no dev — feedback visual imediato
