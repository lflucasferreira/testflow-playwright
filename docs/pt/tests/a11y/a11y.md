# Acessibilidade — Scans axe

**Arquivo fonte:** [`a11y.spec.ts`](../../../../tests/a11y/a11y.spec.ts)

---

## Objetivo

Esta suite valida **acessibilidade** em páginas-chave do TestFlow usando `@axe-core/playwright`. Ela cobre:

- Scans axe de página inteira em Dashboard, Login, Wizard, Components, Settings e UI States
- Scan escopado no formulário Settings (`[data-testid="settings-form"]`)
- Scan específico de modal quando o diálogo Components está aberto
- Filtragem de violações críticas (regra color-contrast desabilitada por limitações do sandbox)

Nenhuma violação crítica axe deve ser reportada no escopo configurado.

## Conceitos do Playwright

| Conceito | Uso neste arquivo |
|----------|-------------------|
| [**`AxeBuilder`**](https://github.com/dequelabs/axe-core-npm/tree/develop/packages/playwright) | Executa análise axe em uma página Playwright |
| **`disableRules(['color-contrast'])`** | Ignora problemas conhecidos de contraste do sandbox |
| **`builder.include(scope)`** | Limita scan a uma subárvore CSS |
| [`loginViaApi`](../../../../support/auth.ts) | Autentica antes de visitar páginas protegidas |
| [`visitAuthenticated`](../../../../support/auth.ts) | Usado no teste de modal Components |
| **Testes parametrizados** | Array `A11Y_PAGES` gera um teste por página |

## Passo a passo — bloco a bloco

### Bloco 1 — Lista de páginas e helper

```typescript
import AxeBuilder from '@axe-core/playwright'
import { test, expect } from '@playwright/test'
import { loginViaApi, visitAuthenticated } from '../../support/auth'
import { ComponentsPage } from '../../pages/ComponentsPage'

const A11Y_PAGES: Array<{
  path: string
  testId: string
  name: string
  scope?: string
}> = [
  { path: '/web/dashboard.html', testId: 'page-dashboard', name: 'Dashboard' },
  { path: '/web/login.html', testId: 'login-email', name: 'Login' },
  // ... Wizard, Components, Settings (scoped), UI States
]

async function assertNoCriticalViolations(page: import('@playwright/test').Page, scope?: string) {
  let builder = new AxeBuilder({ page }).disableRules(['color-contrast'])
  if (scope) builder = builder.include(scope)

  const results = await builder.analyze()
  const critical = results.violations.filter((v) => v.impact === 'critical')
  expect(critical, JSON.stringify(critical, null, 2)).toEqual([])
}
```

- **Dado:** uma lista de páginas com seletores de escopo opcionais.
- **Quando:** `assertNoCriticalViolations` executa axe com color-contrast desabilitado.
- **Então:** zero violações críticas são permitidas.

### Bloco 2 — Scans por página

```typescript
test.describe('Accessibility — axe scans', { tag: ['@a11y', '@regression'] }, () => {
  for (const { path, testId, name, scope } of A11Y_PAGES) {
    test(`${name} has no critical axe violations`, async ({ page, request }) => {
      if (path.includes('login')) {
        await page.goto(path)
      } else {
        await loginViaApi(page, request)
        await page.goto(path)
      }

      await expect(page.getByTestId(testId)).toBeVisible()
      await assertNoCriticalViolations(page, scope)
    })
  }
```

- **Dado:** cada página em `A11Y_PAGES`.
- **Quando:** a página carrega (login não autenticado, demais via auth API).
- **Então:** axe não reporta violações críticas na página inteira ou subárvore escopada.

**Escopo Settings:** apenas `[data-testid="settings-form"]` é escaneado para evitar ruído de outras partes da página.

---

### Bloco 3 — Scan do modal Components

```typescript
  test('Components modal has no critical axe violations when open', async ({ page, request }) => {
    await visitAuthenticated(page, request, '/web/components.html')
    const components = new ComponentsPage(page)
    await components.openModal()
    await assertNoCriticalViolations(page, '[data-testid="modal-overlay"]')
  })
})
```

- **Dado:** a página Components com modal aberto.
- **Quando:** axe escaneia apenas a subárvore do overlay do modal.
- **Então:** ARIA do diálogo e gerenciamento de foco passam nas verificações críticas.
